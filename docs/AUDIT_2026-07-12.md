## 1. Fix symlink escape in the remediation executor's file walker
**ID:** F002 · **Area:** Filesystem

### Improvement
Fix symlink escape in the remediation executor's file walker
### Category
Security
### Current Issue
app/src/lib/agents/executor.ts's walkCode (~lines 24-41) uses statSync/readdirSync which follow symlinks with no realpath containment check. A malicious git repo can commit a tracked symlink pointing outside the clone (e.g. to /etc/passwd or another app path); readFileSync/writeFileSync then read/write THROUGH the symlink to the real target, directly contradicting the code's own comment that 'the user's original source is never mutated.' Matched fragments also leak into the returned FixResult JSON as an arbitrary-file-read-fragment oracle.
### Recommended Solution
Use lstatSync to detect symlinks and skip them, or resolve fs.realpathSync(full) and verify it stays within the sandboxed root before including a file in the walk.
### Expected Benefits
Restores the sandbox invariant; eliminates an arbitrary file read/write primitive reachable by indexing any attacker-controlled repo URL.

Bind fix/route.ts's githubToken to the authenticated session (confused-deputy push relay)
